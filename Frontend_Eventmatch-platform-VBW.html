API Console
pretix REST v1 + Cal.com v2
1
Deploy pretix via Docker
Use the official image: pretix/standalone:stable. Configure PostgreSQL 14+, Redis 6+, Nginx as reverse proxy. Set PRETIX_SITE_URL=https://tickets.yourdomain.com.
2
Generate API token
Admin → Your Organiser → API Keys → Create. Required scopes: events:read events:write orders:read orders:write checkin:write items:write quotas:write questions:write.
3
Create organiser & event
POST to /api/v1/organizers/ then /api/v1/organizers/{slug}/events/ with currency: EUR, timezone: Europe/Vienna, date_from, date_to.
4
Add ticket items + quotas
POST to /items/ for each ticket type. Then POST to /quotas/ to cap availability per item. Link quota to item via items: [id].
5
Add profile questions
POST to /questions/ for company, title, interests. Set required: true for matchmaking fields. Types: S string, T text, C choice, M multi-choice.
6
Enable Stripe payments
Install plugin pretix.plugins.stripe. Set PRETIX_STRIPE_PUBLISHABLE_KEY and PRETIX_STRIPE_SECRET_KEY. Supports SEPA, cards, iDEAL.
7
Configure webhooks
Admin → Webhooks → Add. Listen to pretix.event.order.paid, pretix.event.order.refunded, pretix.event.checkin to trigger Cal.com user provisioning and profile sync.
1
Self-host Cal.com
Clone github.com/calcom/cal.com. Run docker compose up with DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL=https://cal.yourdomain.com. Uses Next.js 14 + PostgreSQL.
2
Create API key
Cal.com Settings → Developer → API Keys → Add. Key format: cal_live_xxxxxxxx. Pass as Authorization: Bearer {key} on all v2 requests.
3
Create event type for meetings
POST to /v2/event-types with length: 30, title: "30-min B2B Meeting", slug: "b2b-meeting", schedulingType: ROUND_ROBIN.
4
Set availability schedules
POST to /v2/schedules with event day slots. E.g. Apr 14: 09:00–12:00 and 13:00–17:00. Block busy periods from pretix session schedule.
5
Create bookings via API
When attendee A requests meeting with B, POST to /v2/bookings with eventTypeId, start, attendee email + name + timeZone. Cal.com handles conflict resolution.
6
Subscribe to Cal.com webhooks
POST to /v2/webhooks. Listen to BOOKING_CREATED, BOOKING_CANCELLED, BOOKING_RESCHEDULED to update meeting slots in real time.
1
On pretix order.paid → create Cal.com user
Webhook handler: receive pretix.event.order.paid, extract attendee email + answers, POST to Cal.com /v2/users to provision their scheduling account.
2
Sync attendee profile to matchmaking DB
Insert attendee record into EventMatch PostgreSQL. Run TF-IDF similarity on interests + industry + looking_for. Recompute top-10 matches per attendee.
3
Meeting request flow
Attendee A clicks request → EventMatch calls Cal.com GET /v2/slots for B → shows slots → on confirm, POST /v2/bookings → update slot status in EventMatch UI.
4
Check-in gate integration
On pretix check-in (PATCH /checkinlists/{id}/positions/{secret}/redeem/), activate the attendee's Cal.com account so slots become bookable to other checked-in attendees only.
